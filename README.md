# ClawNet

> 不要裁判，多派几个选手。同机同模型同输入 → 同一 AST 哈希 → 共识达成。

基于 **AST 归一化 + 多节点哈希比对** 的分布式 AI 验证协议。

## 核心思路

LLM 输出不可靠的根本原因：你不知道它什么时候会胡说。现有方案（LLM-as-a-Judge、Self-Consistency、RLHF）本质上都在用 AI 验证 AI——递归问题。

ClawNet 换了一条路：**不做推理，做验证。**

- 把同一个任务分发给 N 个独立节点
- 每个节点用自己的模型独立推理
- 输出经过 AST 归一化，得到结构无关的确定性哈希
- M of N 哈希一致 → 共识达成

**没有裁判。只有哈希比对。**

## 已验证

| 场景 | 结果 |
|------|------|
| 同一代码，不同格式 | ✅ 哈希一致 |
| 同一逻辑，变量名不同 | ✅ 哈希一致 |
| 同一逻辑，函数/参数全不同 | ✅ 哈希一致 |
| 逻辑不同（恶意/异常节点） | ✅ 哈希不一致，被检出 |
| **同机同模型同输入** | **✅ 归一化哈希一致** |

支持 Go / Python / TypeScript 三种语言 AST 归一化。
