# ClawNet 架构

## 概述

ClawNet 是一个纯对等的分布式算力共享网络。它的设计目标是让任何人都可以用自己闲置的设备加入一个全球性的算力池。

核心思想：**没有中央节点，只有对等协议。**

---

## 网络拓扑

```
          ┌───────────┐
          │  节点 A    │
          │ (MacBook) │
          └─────┬─────┘
                │ gossip
    ┌───────────┼───────────┐
    │           │           │
    ▼           ▼           ▼
┌───────┐ ┌───────┐ ┌───────┐
│ 节点 B │◄───────►│ 节点 C │◄───────►│ 节点 D │
│ (手机) │ gossip  │ (PC)  │ gossip  │ (平板) │
└───────┘         └───────┘         └───────┘
    │                                      │
    │          直连（任务数据）               │
    └──────────────────────────────────────┘
```

ClawNet 节点之间通过两种方式通信：

1. **Gossip 通道** — 用于交换元信息（节点上线/下线、新任务公告、令牌交易）
2. **直连通道** — 用于传输任务数据（帧渲染结果、模型分片）

## 节点内部结构

每个 ClawNet 节点包含以下模块：

```
┌──────────────────────────────────────┐
│              ClawNet 节点              │
│                                        │
│  ┌─────────┐  ┌──────────┐            │
│  │ Gossip  │  │ CRDT     │            │
│  │ 引擎    │◄─┤ 令牌台账  │            │
│  └────┬────┘  └──────────┘            │
│       │         │                      │
│       ▼         ▼                      │
│  ┌─────────────────┐  ┌───────────┐   │
│  │   调度器         │  │ 工人      │   │
│  │  (拆任务/派任务)  │  │ (执行器)  │   │
│  └────────┬────────┘  └─────┬─────┘   │
│           │                   │         │
│           ▼                   ▼         │
│  ┌──────────────────────────────────┐  │
│  │           传输层                   │  │
│  │    (libp2p / 直连 / NAT穿越)      │  │
│  └──────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 1. Gossip 引擎

基于 SWIM 协议（Scalable Weakly-consistent Infection-style Membership）：

- **节点发现**：新节点通过种子节点或局域网广播加入
- **心跳**：定期交换，探测节点存活
- **状态传播**：新任务、新令牌交易全网广播
- **故障检测**：疑似下线 → 间接探测 → 确认下线

### 2. CRDT 令牌台账

使用无冲突可合并数据类型（Conflict-free Replicated Data Type）：

- 每笔令牌交易由双方签名（A帮B跑任务 → B签名确认）
- 每个节点独立维护一份完整的台账副本
- 节点同步时自动合并：CRDT 保证所有副本最终一致
- 不需要共识机制，不需要区块链

### 3. 调度器

每个节点同时是调度器：

- 接收任务公告 → 判断自己的资源是否匹配
- 匹配 → 认领任务 → 交给工人执行
- 不匹配 → 转发给邻居

### 4. 工人（执行器）

可插拔的任务执行模块：

| 模块 | 状态 | 描述 |
|---|---|---|
| `worker-dom` | ✅ 可用 | 浏览器DOM操作（搜索、内容提取、精炼摘要） |
| `worker-image-gen` | 🚧 开发中 | AI图片生成（利用本地显卡/神经引擎） |
| `worker-video-gen` | 🔮 规划 | 分布式视频渲染（帧级别并行） |
| `worker-inference` | 🔮 规划 | 通用大模型推理（MoE 分布式） |
| `worker-simulation` | 🔮 规划 | 物理模拟（FEM网格分解） |

### 5. 传输层

- **小数据（元信息/令牌）**：走 Gossip 通道
- **大数据（视频帧/模型分片）**：走直连通道（libp2p / WebRTC / TCP 直连）
- **防火墙穿越**：STUN/TURN 或 中继

## 分布式视频生成协议

这是 ClawNet 最复杂的任务类型。协议分为三层：

### 层1：潜空间骨架（任务发起者承担）

发起者用自己的设备（不要求高性能）生成一个低分辨率潜空间序列：

```
输入：prompt + 持续时间
输出：64×64 latent 序列（整段视频的运动矢量场 + 构图骨架）
大小：~几 KB / 每秒
模型：轻量化视频扩散（~1-3B 参数，量化后 <2GB）
```

### 层2：帧级别展开（网络贡献者承担）

```
骨架广播到网络
  ↓
贡献者认领帧区间（如 帧25-55）
  ↓
下载骨架 + 目标帧数 + 前后重叠帧
  ↓
本地用 ControlNet 条件放大：
  骨架（低频）→ 作为 ControlNet 输入 → 生成高清帧（高频细节）
  ↓
回传帧序列给发起者
```

每帧独立渲染，天然可并行。重叠帧设计保证拼接时画面连贯。

### 层3：拼接后处理（发起者承担）

```
收到所有节点的帧序列
  ↓
重叠帧渐变融合
  ↓
光流平滑（可选，轻量）
  ↓
编码输出最终视频
```

## 令牌系统

不是货币，不是代币，不是积分。是网络遵循的唯一**调度优先级凭证**。

```python
class TokenLedger:
    """
    每个节点独立维护的 CRDT 台账。
    不需要全局共识，靠签名 + CRDT 保证最终一致性。
    """

    def calculate_priority(self, node_id):
        """
        调度优先级 = 令牌余额 + 在线时常加权
        """
        balance = self.get_balance(node_id)
        uptime  = self.get_uptime(node_id)
        return balance * 0.7 + uptime * 0.3

    def settle_transaction(self, from_node, to_node, amount, signature):
        """
        每笔交易：
        - from_node = 提供算力的节点
        - to_node = 获得服务的节点
        - amount = 消耗的令牌数
        - signature = to_node 签名确认
        """
        verify(from_node, to_node, amount, signature)
        self.apply_crdt(from_node, -amount, to_node, +amount)
```

### 令牌规则

| 操作 | 令牌变化 |
|---|---|
| 加入网络 | +10（新节点启动令牌） |
| 在线1小时 | +1 |
| 帮跑一个标准任务 | +2 |
| 帮跑一个视频渲染任务 | +5 |
| 提交一个标准任务 | -1 至 -3（按算力当量） |
| 提交一个重任务 | -5 至 -10 |
| 举报恶意行为（经验证） | +3 |
| 恶意行为确认 | 清零 + 封禁 |

## 安全性

### 信任模型

ClawNet 采用的信任模型是**累积信任**：

- 新节点可以加入，但优先级低（需要先贡献才能获得调度）
- 每次成功完成任务 → 信任积分 +1
- 每次失败/超时 → 信任积分 -2
- 连续失败 → 被邻居节点标记为不可用

### 任务验证（轻量）

- 视频任务：随机抽检（发起者随机选10%的帧，派另一个节点验证）
- 搜索任务：结果交叉验证（多节点搜索同一关键词，比对结果一致性）
- 恶意行为举证：带有篡改证据的被标记节点全网广播

### 隐私

- 任务数据只在提交者和执行者之间传递
- 不经过任何中继服务器
- 本地推理不出设备
- 令牌交易只包含节点ID和签名，不包含真实身份
